What's wrong with your orchestrator?

⚠️

Dag Parsing = Code Execution

Airflow imports and executes every Python file in your dags/ folder just to parse the DAG structure. A bug in any file brings down the scheduler.

💥

No Safe Deployments

Deploy a bad DAG change and your production pipeline goes down. No rollback. No staging. Just hope and restart the scheduler.

🧪

Testing is Expensive

Testing pipeline changes requires a full staging environment. Full Airflow cluster, database, message broker. Copy of production data. Cost and latency killed.

⏱️

Polling is Slow

The scheduler polls your database every second, checking if tasks are ready. Latency adds up. Consistency is a guess.

🛠️

Dependency Hell

PostgreSQL. Redis. Celery. Gunicorn. A webserver. Just to say "hello world". Your orchestrator has become infrastructure.

🚀

Scaling is Manual

Airflow scales horizontally through Celery and message brokers. Complex. Fragile. Needs constant tuning and monitoring.

What if your orchestrator was designed today?

Compile, Don't Execute

Tree-sitter parses Python AST without running it. Your DAGs are analyzed, not executed. No side effects. No crashes during parsing.

Virtual Environments

Test in prod. Safely.

Get started in 60 seconds

Install

curl -sSL https://install.conduit.dev | sh

One-liner. Grab the binary. That's it.

Initialize

conduit init my_project && cd my_project

Create a new project directory with templates.

Write a DAG

cat > dags/hello.py << 'EOF' from conduit import DAG, Task dag = DAG("hello_world") with dag: say_hello = Task("hello") EOF

Define your pipeline in Python.

Compile

conduit compile

Parse and validate your DAGs. Takes milliseconds.

Run Locally

conduit run hello_world

Test your pipeline without needing infrastructure.

Serve Dashboard

conduit serve

Start the server. Open http://localhost:9000
