================================================================================
KEY DESIGN DECISIONS (v0.1)
================================================================================

1. JSON INTERCHANGE
   - Complex types serialized as JSON strings
   - Advantages: Easy debugging, language-agnostic, extensible
   - Pragmatic for v0.1; can optimize to native classes in v0.2+

2. ERROR HANDLING
   - All Rust ConduitError converted to Python ValueError
   - Error messages preserved for debugging
   - Consistent error handling across all modules

3. MODULE ORGANIZATION
   - Four submodules mirror underlying Rust crates
   - Each module is independent but composable
   - Stateless functions + EnvironmentStore for state

4. COMPILATION
   - Uses PyO3 0.22 with extension-module feature
   - Built via maturin (modern Python/Rust build tool)
   - Native compiled module (cdylib) for performance

================================================================================
USAGE EXAMPLE
================================================================================

from conduit_native import compiler, planner
from conduit_native.state import EnvironmentStore

# 1. Compile DAGs
plan_json = compiler.compile_dags("/path/to/dags")

# 2. Compute fingerprints
fps_json = planner.compute_fingerprints(plan_json)

# 3. Manage environments
store = EnvironmentStore("/state")
store.create_env("prod")
store.promote("dev", "prod")
store.save()

# 4. Detect changes
env_json = store.get_env("prod")
changes_json = planner.detect_changes(plan_json, env_json)

# 5. Analyze lineage
lineage_json = lineage.extract_sql_lineage("SELECT col FROM table")
