                         CONFIGURATION SUPPORT
================================================================================

S3 Configuration Fields:
  Required:
    • database               - S3 bucket name

  Optional:
    • region               - AWS region (defaults to us-east-1)
    • prefix               - Key prefix (defaults to empty)
    • access_key_id        - AWS access key (optional)
    • credentials          - AWS secret key (optional)
    • endpoint_url         - Custom S3 endpoint (MinIO, etc.)
    • host                 - Alternative endpoint specification

GCS Configuration Fields:
  Required:
    • database             - GCS bucket name

  Optional:
    • project              - GCP project ID
    • prefix               - Object prefix (defaults to empty)
    • credentials          - Service account JSON path or env var
    • extra.project        - Alternative project specification

================================================================================
                       AUTHENTICATION METHODS
================================================================================

S3 Authentication:
  1. Static Credentials
     - access_key_id + credentials (secret access key)
     - Resolved via resolve_credential() from env vars or literals

  2. AWS Default Credential Chain
     - Environment variables (AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY)
     - ~/.aws/credentials file
     - IAM role (EC2, ECS, Lambda)
     - STS temporary credentials

GCS Authentication:
  1. Service Account JSON
     - File path via credentials: file:///path/to/sa.json
     - GOOGLE_APPLICATION_CREDENTIALS environment variable

  2. Application Default Credentials (ADC)
     - ~/.config/gcloud/application_default_credentials.json
     - %APPDATA%/gcloud/... (Windows)
     - Cloud Run, GKE workload identity, Cloud Shell, etc.
